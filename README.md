# PRD for AI Products

> 专为AI应用产品编写的PRD生成指令包Skill。
> 通过15个决策维度的系统化提问，输出结构完整、可落地的专业AI产品PRD。

**通用AI指令** · 零外部依赖 · 适用于任何支持Markdown指令的AI平台

---

## 一句话

当你对AI产品有了模糊想法，这个指令不会让你立刻开始写PRD——它会先用**5批提问、每批3个维度（每维度展开若干子问题）**，逐一锁定15个核心决策维度。全部确认后，才输出一份15章结构的完整PRD。

---

## 项目背景与方法论来源

目标是把 AI 产品 PRD 的方法论**产品化、可复用化**——让"从模糊想法到专业 PRD"这一过程可被任何 AI 平台稳定执行。

- **方法论**：含 Agent PRD 八大基础模块、六个高频坑、AI 交互六模式 / 五坑、传统 vs AI PRD 三差异
- **设计借鉴**：公开标杆 skill 的成熟机制（复杂度分级、三分法、速读卡、方法论外显等），已在下方「参考与致谢」逐一致谢。

## 为什么AI产品需要专属PRD

| | 通用PRD | AI产品PRD（本指令） |
|--|--------|-------------------|
| 提问维度 | 通用维度 | 15个中**6个替换为AI专属** + **AI深度增强维度（条件触发）** |
| PRD章节 | -- | **15章 + 4个AI专属章 + AI深度增强子章** |
| 竞品对标 | 功能/市场 | 增加**AI能力维度对标** |
| 商业分析 | 商业模型 | 增加**算力经济学** |
| 质量可判定 | -- | 交付前**PRD质量星级自评（★★★★★）** |

传统PRD回答不了AI产品的核心问题：**模型怎么选？能力边界在哪？算力成本怎么算？数据飞轮怎么转？AI幻觉怎么办？评估怎么验收？** 这些才是AI PM的真正考题。

> 英文代号 **write-ai-prd**，便于在 Agent 技能市场检索与引用。

## 工作流程

```
Step 1  接收构思 ──────────── 理解用户的AI产品模糊想法
        │
Step 2  范围定级 + 分5批提问 ─ 先定级 L1–L4 并让用户决策是否向上取（L4 直接对标），再每批3问锁定15核心维度
        │
Step 3  AI深度增强维度 ─────── 条件触发：评估验收/Agent行为/工具接口/交互模式
        │
Step 4  编写PRD ───────────── 含4个AI专属章节的15章结构
        │
Step 5  四重校验 + 质量自评 ─ 决策点覆盖 + ★★★★★自评 + 缺口扫描
        │
Step 6  交付 ──────────────── 完整PRD + 双维度亮点提炼
```

## 15 个决策维度

### 通用维度（9个）

| 批次 | 维度 | 核心决策 |
|:----:|------|---------|
| 1 | 产品定位 | 这到底是什么产品？ |
| 1 | 目标用户 | 先打动谁？用户分层权重？ |
| 1 | 优先市场 | 先打哪？国内/海外/同步？ |
| 2 | 命名与品牌 | 叫什么？占位代号+命名逻辑 |
| 2 | 平台形态 | 从哪进？主战场/裂变入口/后续扩充 |
| 3 | 商业化模型 | 怎么赚钱？免费层/付费层/社交货币转化 |
| 4 | 社区与社交 | 除了核心流程还有什么齿轮？ |
| 5 | 竞品锚定 | 对标谁？差异化说服力来源 |
| 5 | PRD呈现定位 | 写给谁看？立项/实操/应聘 |

### AI 专属维度（6个）🤖

| 批次 | 维度 | 核心决策 |
|:----:|------|---------|
| 1 | **AI产品形态** 🤖 | AI原生/AI增强/AI基础设施/AI+领域融合 |
| 2 | **AI能力边界** 🤖 | 能力范围/质量下限/可控性/延迟容忍/渐进增强 |
| 3 | **AI技术栈与模型策略** 🤖 | 模型来源/类型/更新/部署/降级方案 |
| 3 | **数据飞轮** 🤖 | 数据收集/反馈闭环/自增强/合规 |
| 4 | **AI UX范式** 🤖 | 对话式/指令式/协作式/嵌入式/Agent |
| 4 | **AI安全伦理与合规** 🤖 | 幻觉应对/版权/隐私/偏见/监管 |

## AI 深度增强维度（条件触发）

15个核心维度是骨架，以下维度按产品类型**条件触发**，补全"Agent PRD八大块"缺口：

| 触发条件 | 增强维度 | 必要性 |
|---------|---------|-------|
| 所有AI产品 | 评估与验收策略 | 必答——概率输出必须靠评估衡量 |
| 所有AI产品 | 六个关键交互模式 | 必答——不确定/多轮/等待/出错 |
| Agent类产品 | Agent行为定义（场景→期望行为） | 必答 |
| Agent类产品 | 工具与接口（RAG/FC/MCP） | 必答——读放手/写设防 |
| 生成/知识类 | Prompt规格 + 知识源与更新 | 必答——Prompt即功能实现 |

## PRD 章节结构（15章）

```
一、产品概述
二、市场分析与竞品洞察（含AI能力维度对标）
三、产品定位与核心策略
   └─ 3.2 AI能力边界定义                    ← AI专属
四、目标用户
五、产品架构与功能模块
六、AI UX范式与交互设计                      ← AI专属
七、AI技术栈与模型策略                        ← AI专属
八、数据飞轮与AI自增强                        ← AI专属
九、商业模型（含算力成本结构）
十、社区与社交生态
十一、AI安全、伦理与合规                      ← AI专属
十二、技术架构（含AI工程深度）
十三、版本迭代规划
十四、成功指标与KPI
十五、风险评估
```

## 文件结构

> **关键约定**：`README.md` 仅作仓库说明，**不被 skill 加载**；实际加载的是内层 `prd-for-ai-products/` 目录（SKILL.md + references/）。

```
prd-for-ai-products/                  ← 仓库根目录（本文件所在层）
├── README.md                          # 仓库说明，不随 skill 加载
└── prd-for-ai-products/               ← 实际 skill 包，AI 平台加载此目录
    ├── SKILL.md                       # 主指令（6步流程 + ★★★★★质量自评）
    │   ├── 前置元数据（name/description/triggers）
    │   ├── 15核心维度 + AI深度增强维度（条件触发）
    │   └── 2个精简示例（AI原生 vs AI增强）
    └── references/
        ├── ai-decision-framework.md   # 15核心+AI深度增强维度完整提问框架
        │   ├── 5批×3问的维度定义与选项模板
        │   ├── AI深度增强维度（评估验收/交互模式/Agent行为/工具接口/Prompt）
        │   └── 提问节奏规范（7条）
        └── ai-prd-template.md         # 15章 + AI深度增强章 + Agent条件章
            ├── 标准章节骨架（含4个AI专属章节标记）
            ├── 13条撰写原则 + 写作机制
    └── dist/
        └── create-prd-for-ai-products-universal-prompt.md  # 单文件通用版（零依赖，可粘贴任意AI）
```

### 安装方式

- **方式 A · 技能包（完整版，推荐）**：将内层 `prd-for-ai-products/` 目录放入 AI IDE 的 `.cursor/skills/`（或对应 skills 目录）；网页端把 `SKILL.md` 全文作为指令，references 按需附上。完整版含 5 批提问模板、决策框架与选项库。
- **方式 B · 单文件通用版（零门槛）**：不想折腾目录？直接把 `prd-for-ai-products/dist/create-prd-for-ai-products-universal-prompt.md` **全文复制粘贴**给任意 AI（ChatGPT / Claude / 通义 / 豆包 / Cursor / Claude Code），即可使用——自包含、无外部依赖。
- **通用原则**：本指令零外部依赖、无平台锁定，任何能读 Markdown 的 AI 都能执行。

## 示例触发

### 输入

> "我想用xxx的IP资产做一款AI即时生成小游戏的应用，竞品没有这些IP。"

### 过程

1. 识别为 **AI原生产品**
2. 分5批提问，每批锁定3个维度
3. 条件触发AI深度增强维度（评估验收/Prompt规格/生成质量评估）
4. 输出15章PRD
5. 四重校验 + ★★★★★质量自评：决策点覆盖 ✅ + 4维全5★

### 输出亮点（示例）

- **产品亮点**：IP即护城河 / Remix飞轮模型 / 算力分级计次体系
- **AI 特性亮点**：AI能力边界的产品化表述 / 算力经济学量化分析 / 渐进式AI能力路线设计

## 质量星级自评（★★★★★）

交付前，指令会对PRD做4维星级自评并展示给你——让"质量"从主观感受变为可判定信号（借鉴成熟技能的可判定质量机制）：

| 维度 | 看什么 |
|------|--------|
| 决策完整度 | 15核心 + 增强维度是否全锁定 |
| AI能力深度 | 边界/算力/飞轮/评估/安全是否全到位 |
| 可执行性 | 行为+边界+兜底+评估集是否可交付 |
| 风险可控性 | 异常降级+安全合规是否闭环 |

任一维度低于3★先修订再交付。

## 安全边界

本指令只做「提问 + 生成 PRD」，并守住以下边界：

- **不替你决策**：只引导、给推荐并说明理由，拍板权在你。
- **不编造**：信息缺失标 `[TODO]`、不确定标 `[未知]`，交付前汇总，绝不虚构数字。
- **不执行危险操作**：不删文件、不提交 git、不发外部请求。
- **不越权**：仅在你给的 AI 产品构思内生成；若实为传统非 AI 产品，会建议改用通用 PRD 流程。
- **可中断**：任一步都可喊停、改方向、重来。
